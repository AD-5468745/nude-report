# 누드TV 일일 리포트 자동 발송

매일 정해진 시간에 구글 시트 데이터를 읽어 **이미지 2장**(월 누적 · 업체별)을 만들고
**전체 업체 목록(스포일러)**과 함께 텔레그램 채널로 자동 발송합니다.
내 컴퓨터가 꺼져 있어도 GitHub 서버에서 실행되며, 무료입니다.

---

## 1. 구글 시트 준비

탭 2개를 만들고, 시트를 **"링크가 있는 모든 사용자 - 뷰어"**로 공유하세요.

**탭 ① `가입기록`** — 가입이 들어올 때마다 한 줄씩 추가
| 날짜 | 업체명 | 신규가입수 |
|------|--------|-----------|
| 2026-06-07 | 판도라 | 4 |
| 2026-06-07 | 솔카지노 | 5 |

- 날짜는 `YYYY-MM-DD` 형식 (예: 2026-06-07)
- 1행은 헤더(제목 줄)

**탭 ② `업체목록`** — 전체 업체를 표시할 순서대로
| 업체명 |
|--------|
| 판도라 |
| 네임드 |
| … |

> 캡션 "전체 업체 펼쳐보기"는 이 `업체목록` 순서 그대로 나옵니다(가입 0인 곳 포함).

**시트 주소에서 값 2개 확인:**
`https://docs.google.com/spreadsheets/d/`**`여기가_SHEET_ID`**`/edit#gid=`**`여기가_GID`**
- 각 탭을 클릭하면 주소 끝 `gid=` 숫자가 바뀝니다 → `가입기록` gid, `업체목록` gid를 각각 메모.

---

## 2. 텔레그램 봇 만들기

1. 텔레그램에서 **@BotFather** 검색 → `/newbot` → 안내대로 만들면 **봇 토큰**을 줍니다.
2. 발송할 **채널**에 이 봇을 **관리자로 추가**(게시 권한 필요).
3. 채널 ID 확인:
   - 공개 채널: `@채널아이디` 그대로 사용
   - 비공개 채널: 채널에 아무 글이나 올린 뒤 `https://api.telegram.org/bot<봇토큰>/getUpdates` 열어서 `"chat":{"id":-100...}` 숫자 확인

---

## 3. GitHub 저장소에 올리기

1. GitHub에서 새 저장소 생성.
   - **공개(public)**: Actions 실행시간 무제한 무료
   - 비공개(private): 월 2,000분 무료 — 이 작업은 하루 1~2분이라 충분
2. 이 폴더의 파일을 그대로 업로드:
   ```
   generate_and_send.py
   report_template.html
   requirements.txt
   .github/workflows/month-report.yml      # 월 누적(일일별) — 낮 12시
   .github/workflows/company-report.yml    # 업체별 + 스포일러 — 저녁 8시
   ```

---

## 4. 비밀값(Secrets) 등록

저장소 → **Settings → Secrets and variables → Actions → New repository secret** 에서 5개 등록:

| 이름 | 값 |
|------|----|
| `SHEET_ID` | 시트 ID |
| `GID_RECORDS` | `가입기록` 탭 gid |
| `GID_COMPANIES` | `업체목록` 탭 gid |
| `BOT_TOKEN` | BotFather가 준 봇 토큰 |
| `CHANNEL_ID` | `@채널아이디` 또는 `-100...` |

> 토큰·ID는 코드에 직접 넣지 말고 반드시 여기 Secrets에만 넣으세요.

---

## 5. 발송 시간 (이미 설정됨)

리포트가 **2개의 워크플로우로 분리**되어 서로 다른 시간에 발송됩니다(둘 다 **전날** 기준). `cron`은 **UTC 기준**이라 한국시간 −9시간:

| 워크플로우 | 내용 | 발송(KST) | cron(UTC) |
|-----------|------|-----------|-----------|
| `month-report.yml` | 월 누적(일일별) 이미지 | 낮 12:00 | `0 3 * * *` |
| `company-report.yml` | 업체별 이미지 + 전체 업체 스포일러 | 저녁 20:00 | `0 11 * * *` |

> 시간을 바꾸려면 해당 `.yml`의 `cron` 값을 (원하는 KST −9시간)으로 수정하세요.

> 스크립트는 `python generate_and_send.py month`(월 누적만) / `day`(업체별+스포일러만) / `all`(전부, 수동테스트용)로 동작합니다.

---

## 6. 테스트 & 동작

- 저장소 **Actions** 탭 → `nudeTV-month-report` 또는 `nudeTV-company-report` → **Run workflow**(수동 실행)로 즉시 테스트.
- 이후 매일 지정 시간에 자동 실행. 리포트는 **전날** 기준입니다.

---

## 참고

- **세로로 길어지면 자동 다단**: 업체/일수가 많아지면 2~4단으로 펼쳐져 한 장에 깔끔하게 들어갑니다.
- **월 자동 리셋**: 날짜 기준으로 그달치만 집계하므로 달이 바뀌면 알아서 새로 시작합니다.
- **이미지 선명도**: 더 또렷하게 하려면 `generate_and_send.py`의 `SCALE = 2` 를 `3`으로.
- **X 로고 교체**: 공식 `.svg`를 받으면 템플릿의 X 부분만 바꾸면 됩니다.
- **주의**: GitHub Actions 예약 실행은 저장소에 **60일간 커밋이 없으면 자동 비활성화**됩니다. 가끔 커밋하거나 수동 실행하면 유지됩니다. (예약 시간이 서버 부하로 수 분~수십 분 늦을 수 있으나 일일 리포트엔 문제없습니다.)
